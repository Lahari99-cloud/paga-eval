# Institutional Deployment Guide

`paga-eval` is an auditable evaluation guardrail for child-reading tutor agents.
It is not a clinical diagnostic system and does not replace educator or
speech-language pathologist judgment.

## Required Pre-Deployment Review

Before using a policy pack with children:

1. Have an educator and speech-language pathologist approve each rule, locale,
   age range, rationale, and intervention policy.
2. Configure a versioned `PolicyPack` per approved population. Do not treat the
   bundled `en-US` starter rules as a clinically validated universal policy.
3. Run a representative benchmark across approved cohort dimensions. Review
   over-intervention, under-intervention, and escalation rates.
4. Define a human-review workflow for `ESCALATE_REVIEW`.
5. Complete your organization's privacy, security, accessibility, and legal
   review before processing student data.

## Privacy Boundary

The core evaluator requires only a target word, an attempted reading, and an
agent action. Keep raw audio outside this package unless your approved use case
requires it.

`LearnerProfileAdapter` pseudonymizes learner identifiers by default and supports
profile export, deletion, and retention pruning. In production:

- provide a deployment-specific secret `pseudonymization_salt`;
- set `production_mode=True` to reject the demo salt;
- configure `retention_days` and call `prune_expired_profiles()` from your
  scheduled maintenance path;
- store the salt in your secrets manager;
- apply deletion to any downstream durable stores as well as the in-memory adapter;
- encrypt stored data and enforce role-based access;
- prohibit advertising and secondary model-training use unless separately
  reviewed and authorized;
- record the applicable school authorization or consent workflow.

## Audit And Reporting

Every `evaluate()` call returns an `AuditRecord` with:

- evaluation ID and UTC timestamp;
- policy ID and version;
- omitted target and attempt text by default, plus the normalized action;
- classification, verdict, distances, and applied rule IDs;
- whether human review is required.

Configure `AuditTextMode.HASHED` only when transcript correlation is necessary.
Reading words have low entropy, so unkeyed hashes can still be dictionary-guessed.
Use `AuditTextMode.PLAINTEXT` only after an explicit data-governance review.

Use `audit_records_to_jsonl()` for append-only ingestion and
`build_institutional_report()` for cohort-level quality reporting. Cohort labels
must be approved, broad reporting dimensions. Avoid encoding sensitive
learner-level attributes without a governance process.

## Governed Policy Configuration

`PolicyPack.to_dict()` and `PolicyPack.from_dict()` support reviewable
configuration storage. Policy validation rejects empty identifiers, mismatched
locales, duplicate rule IDs, invalid thresholds, and unbounded rule growth.
Retain approved policy versions so historical audit records remain interpretable.

## Included Integration Contracts

The package exposes typed Python objects, JSON-serializable records, a
transport-neutral `EvaluationService`, an LTI AGS score-body mapper, and a
starter Ed-Fi assessment-result mapper with optional CASE references.

These are integration contracts, not certified hosted adapters. Production
adapters should be implemented and validated per buyer requirements:

- buyer identity-provider integration and authorization policy;
- LMS launch, authentication, and score passback through LTI Advantage;
- approved standards alignment through CASE;
- deployed Ed-Fi version, descriptors, extensions, and API profile;
- accessibility validation for any user interface;
- warehouse ingestion and retention enforcement.

## Hosted API Reference Deployment

Install `paga-eval[service]` or build the included `Dockerfile`. The reference
FastAPI deployment is created by `paga.api:create_app`.

Required secrets:

- `PAGA_API_KEY`: ingress API key with at least 24 characters;
- `PAGA_API_KEYS_JSON`: optional tenant-scoped credential list for production;
- `PAGA_PROFILE_SALT`: independent HMAC salt with at least 24 characters;
- `PAGA_ENCRYPTION_KEY`: Fernet key generated by
  `EncryptedProfileStore.generate_key()`.

Operational configuration:

- `PAGA_DATABASE_PATH`: encrypted-profile and encrypted-audit SQLite path;
- `PAGA_RETENTION_DAYS`: profile and audit retention window;
- `PAGA_MAX_REQUEST_BYTES`: request-size ceiling;
- `PAGA_LOG_REQUESTS`: emit privacy-bounded JSON request logs, default `true`;
- `PAGA_DECRYPTION_KEYS`: comma-separated prior encryption keys during rotation.

The reference service provides API-key authentication, request-size enforcement,
no-store response headers, health and readiness endpoints, encrypted profile
storage, append-only encrypted audit storage, retention pruning, and operational
record counts.

For a single trusted integration, `PAGA_API_KEY` grants every permission to the
default tenant. For multi-tenant environments, omit it and configure
`PAGA_API_KEYS_JSON`:

```json
[
  {
    "key": "load-from-your-secrets-manager",
    "tenant_id": "district-a",
    "permissions": ["evaluate", "report", "profile:read", "profile:write", "operations"]
  }
]
```

Tenant namespaces isolate encrypted profiles, audit replay IDs, retention
maintenance, and operational counts. API responses include `X-Request-ID`; valid
caller-provided IDs are preserved for distributed tracing.

The service emits JSON request logs by default for centralized collection. Each
event contains request ID, tenant namespace when authenticated, method, path,
status, and duration. It intentionally excludes headers, query strings, bodies,
transcripts, learner identifiers, and API keys. Set `PAGA_LOG_REQUESTS=false`
only when an approved ingress emits an equivalent privacy-bounded event.

Use API-key authentication only as the reference boundary. Enterprise deployments
should terminate TLS at an approved ingress, store secrets in a secrets manager,
apply ingress rate limits, forward request IDs, centralize audit events, back up
the encrypted data store according to policy, and integrate the institution's
identity provider and authorization model.

### Encryption-Key Rotation

1. Generate a new primary Fernet key.
2. Deploy it as `PAGA_ENCRYPTION_KEY` while retaining old keys in
   `PAGA_DECRYPTION_KEYS`.
3. Run `python -m paga.maintenance rotate-encryption` in an approved maintenance
   task.
4. Verify stored profiles and audits, then remove old keys.

### Encrypted Store Maintenance

Run maintenance commands with `PAGA_DATABASE_PATH`, `PAGA_ENCRYPTION_KEY`,
`PAGA_DECRYPTION_KEYS`, and `PAGA_RETENTION_DAYS` loaded from the approved
runtime secret source.

```bash
python -m paga.maintenance check
python -m paga.maintenance backup --output /approved-backup-path/paga.sqlite3
python -m paga.maintenance prune
```

`check` validates SQLite structure and decryptability of every encrypted profile
and audit record. `backup` uses SQLite's online backup API, validates the source
and snapshot, refuses accidental replacement unless `--overwrite` is supplied,
and publishes the snapshot atomically. Retain and protect backup encryption keys
according to the institution's recovery policy. Exercise restoration in a
non-production environment before launch and on the institution's recovery-test
schedule.

## Benchmark Boundary

Use `run_benchmark()` with institution-reviewed labeled cases before deployment.
The bundled `benchmarks/starter_en_us.json` fixture exercises package behavior
only. It is not a clinical validation dataset or a substitute for multilingual,
dialect, accessibility, and representative-population evaluation.
